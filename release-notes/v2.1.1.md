# v2.1.1 · New API group-token source update

## Highlights

- Added public extension hooks for local-only provider controls: settings add-ons, node add-ons, runtime hooks, and `providerParams` passthrough for image, video, Seedance, and audio nodes.
- Prepared the New API group-token advanced mode boundary. The public repo keeps only generic hooks; local private group-token code remains outside GitHub and can be encrypted into Electron builds later.
- Kept FAL image/video routes on the common Zhenzhen API key path so group-token selection does not affect FAL fallback models.
- Added explicit clear buttons for each classified API key field in API settings.
- Fixed Nano Banana 2 mapping: the visible `nano-banana-2 (Flash)` option now submits `gemini-3.1-flash-image-preview`, and old saved `nano-banana-2` values are normalized on the frontend/backend path.
- Added invalid-token cache invalidation hooks so private group-token implementations can discard stale keys when users delete and recreate New API tokens.

## Source-Only Boundary

- This version is a source-only GitHub update.
- No Electron installer was built.
- No GitHub Release or `latest.yml` auto-update asset was created or updated.
- Public source continues to exclude `skill.md`, `roadmap.md`, `local-private/**`, and local-only commercial overlay code.
